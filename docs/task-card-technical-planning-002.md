# 线程任务卡

### 线程
技术规划

### 编号
TP-002

### 目标
基于 `TP-001` 的技术结论，收敛“本轮代码实现的最小可落地方案”，明确本轮到底实现到哪一层，避免代码线程继续在范围上摇摆。

### 范围
- 要做：
  - 明确本轮是否只做“参数接口 + 结构化消息行 + 增量修复”
  - 明确本轮是否包含真实 Tier 2 深扫实现
  - 明确默认通道切换到 Android 的实现边界
  - 明确与用例 3 相关的技术处理建议
  - 明确给代码编写线程的模块改动边界
  - 明确验收点和回归风险
- 不要做：
  - 不直接改代码
  - 不修改产品优先级
  - 不扩展到 Android 自动发送
  - 不扩展到 Web 通道恢复
  - 不扩展到多账号、后台、交付系统

- `docs/thread-status.md`

### 输出要求
- 一份“本轮最小实现范围”的明确结论
- 一份推荐采用的单一方案，不只列备选
- 一份给代码编写线程可直接执行的任务拆解
- 一份验收建议，至少覆盖：
  - 默认通道切换
  - 用例 3
  - 回归风险
  - 是否具备进入上传阶段的前置条件

### 完成标准
- 有单一推荐方案
- 有明确 in scope / out of scope
- 有文件级改动边界
- 有测试验收建议
- 结果可以直接作为代码编写线程下一张任务卡输入
